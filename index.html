git学习，分支提交

首次将代码提交到git

1、初始化 Git 仓库：在你的项目文件夹中打开命令行，运行以下命令来初始化一个新的 Git 仓库git init


2、添加文件到暂存区：使用以下命令将你的文件添加到 Git 暂存区（Index）中，准备提交git add .


3、提交文件到 Git 仓库：在命令行中，运行以下命令来提交文件到 Git 仓库中git commit -m "提交信息"


4、关联远程仓库（可选）：如果你计划将代码推送到远程仓库（如 GitHub、GitLab 等），可以使用以下命令将本地仓库与远程仓库关联起来git remote add origin <远程仓库URL>


5、将代码推送到远程仓库（可选）：最后，如果你有关联远程仓库，可以使用以下命令将代码推送到远程仓库：git push -u origin master


如何将新的代码提交到远程仓库？

1、初始化 Git仓库：在你的项目文件夹中打开命令行，运行以下命令来初始化一个新的 Git仓库git init


2、添加文件到暂存区：使用以下命令将你的文件添加到 Git 暂存区（Index）中，准备提交git add .


3、提交文件到 Git 仓库：在命令行中，运行以下命令来提交文件到 Git 仓库中git commit -m "提交信息"


前三步只是将代码传到本地仓库，还要通过后两步将代码传到远程仓库

4、关联远程仓库（可选）：如果你计划将代码推送到远程仓库（如 GitHub、GitLab 等），可以使用以下命令将本地仓库与远程仓库关联起来git remote add origin <远程仓库URL>


5、将代码推送到远程仓库（可选）：最后，如果你有关联远程仓库，可以使用以下命令将代码推送到远程仓库：git push -u origin master


如何创建新的分支？

创建分支：在本地仓库中，使用以下命令创建一个新的分支：git branch branch-name，其中 branch-name 是你要创建的分支名称。

切换到新创建的分支：git checkout branch-name 或 git switch branch-name。


在新分支上进行代码修改在新分支上进行代码修改和开发工作


添加、提交代码使用git add . 命令将修改的文件添加至暂存区;使用 git commit -m "提交信息" 命令提交代码到当前分支。


推送代码至远程仓库：如果远程仓库还没有对应的分支，需要先创建远程分支：git push origin branch-name;接着将本地分支与远程分支关联并推送代码：git push -u origin branch-name。
